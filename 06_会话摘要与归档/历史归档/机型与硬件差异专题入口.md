# 机型与硬件差异专题入口

## 1. 这份文件的定位

这份文件用于承接 TNAS 不同机型、不同架构、不同启动方式和不同内存代际带来的**测试边界差异**。  
它不是产品宣传资料，也不是硬件选型文档，而是为了让后续测试判断时，先知道“当前现象是否可能与机型或硬件差异有关”。

这份专题入口当前主要服务以下场景：

- 安装 / 初始化 / 升级类测试
- 机型兼容性判断
- 启动异常、引导异常、刷机异常分析
- 内存显示、内存上限、内存兼容性问题分析
- 文档、官网参数、页面展示与实机现象的一致性比对

## 3. 为什么这份专题需要单独存在

当前共享底座已经确认：TNAS 不是单一机型产品，而是多机型、多架构、多代际并存的产品体系。  
这意味着很多测试结论都不能脱离硬件背景直接下：

- 同一个功能，可能在 `ARM` 与 `x86` 上支持范围不同
- 同一个安装或启动问题，可能与 `固件引导 / Legacy / UEFI` 差异有关
- 同一个内存显示问题，可能是“厂商标称值”“系统可管理值”“用户可用值”被混写
- 同一个版本问题，可能只影响某一机型系列或某一代硬件

因此后续只要任务涉及机型、架构、启动、内存或兼容性，就不应该只看页面现象，而应该先回到这份专题入口做机型分层。

## 4. 当前已确认的核心差异维度

当前可以先稳定为以下几类差异维度：

- 架构差异：`x86`、`ARM`
- 启动方式差异：`固件引导`、`Legacy`、`UEFI`、`UEFI/Legacy`
- 版本线差异：不同机型对应 `TOS 4 / TOS 5 / TOS 5.1 / TOS 6`
- 内存代际差异：`DDR4`、`DDR5`
- 内存容量差异：自带内存、插槽数、最大支持内存
- 型号系列差异：`22 / 23 / 24 / 450 / 500 / 全闪 / 机架式` 等

## 7. 内存代际与容量差异的当前口径

### 物理内存、系统可管理内存、用户可用内存要分开看

根据 `内存介绍.md` 与 `内存容量计算.md`，后续涉及内存问题时，至少要区分：

- 物理安装的内存条容量
- 操作系统可管理的总内存
- 用户程序实际可用内存
- 缓冲 / 缓存占用后的实时可用量

不能把这些概念混写成一个“总内存”。

### 厂商标称值与系统显示值不能直接等同

当前资料已明确：

- 厂商标称容量常按十进制理解
- Linux 系统中的很多内存显示按二进制体系理解
- 同时还存在硬件保留、内核保留等占用

因此像“标称 4GB，系统里为什么只看到约 3.x GiB”这类现象，不能直接判定为缺陷，必须先分清：

- 标称值口径
- 系统显示单位
- 硬件保留和内核保留

### 当前已确认的内存代际分层

根据 `内存介绍.md` 与型号表，当前可先稳定吸收以下分层：

- 旧机型 `DDR4`：资料中明确提到 `220 / 420 系列、221 / 421 / 422 系列、423 系列、450 系列`
- 新机型 `DDR5`：资料中明确提到 `424 系列、SSD 系列、500 系列`

这条分层与型号表抽样结果是相互印证的：

- `24 系列`、`500 系列`、`全闪系列` 已出现 `DDR5`
- `23 系列`、`450 系列` 已出现 `DDR4`

### 当前已确认的内存槽位和上限示例

根据型号表当前可确认的样例：

- `F6-424 Max`：`8 GB DDR5`，`2` 插槽，最大支持 `64 GB DDR5`
- `F6-424`：`8 GB DDR5`，`1` 插槽，最大支持 `32 GB DDR5`
- `T12-450`：`16 DDR4`，`2` 插槽，最大支持 `32`
- `T9-450`：`8 DDR4`，`2` 插槽，最大支持 `32`
- `T12-500 Pro`：`16 DDR5`，`2` 插槽，最大支持 `64`
- `T9-500 Pro`：`16 DDR5`，`2` 插槽，最大支持 `64`
- `F8 SSD`：`8 GB DDR5`，`1` 插槽，最大支持 `32 GB DDR5`
- `F8 SSD Plus`：`16 GB DDR5`，`1` 插槽，最大支持 `32 GB DDR5`

这里也要保持谨慎：

- 部分型号字段为空，不代表没有插槽或没有上限，只代表当前资料缺失
- 同一个系列也可能存在 `Max / Pro / Plus / 普通版` 的硬件差异，不能串用参数

### 当前已确认的内存通道与混插基础口径

根据 `内存介绍.md`：

- 单通道通常指安装一条内存
- 双通道通常指安装两条内存
- 双通道又存在对称与非对称差异
- 相同容量下，对称双通道通常优于非对称双通道
- 不同代际内存不能混用

### 对测试的直接意义

后续测试至少要重点关注：

- 页面展示内存值和官网标称值是否混淆了单位
- 实机可识别内存、插槽数、最大支持内存是否与文档一致
- DDR4 / DDR5 代际是否被错误混写
- 同系列不同子型号之间是否错误复用了内存参数
- 性能、安装、兼容性问题是否和单通道 / 双通道有关

## 8. 当前可用于验证内存与硬件信息的命令入口

根据当前资料，以下命令可作为后续证据链入口：

- `free -h`
- `dmidecode -t memory`
- `top`
- `ps`
- `cat /proc/<pid>/status`
- `smem`
- `atop`
- `cat /proc/meminfo`
- `dmesg | grep -i available`
- `uname -a`

这些命令在测试里的作用并不相同：

- `free -h`、`/proc/meminfo` 更适合看系统级总量与可用量
- `dmidecode -t memory` 更适合看物理内存类型、插槽、速度、最大容量
- `top`、`ps`、`smem`、`atop` 更适合看进程级占用
- `uname -a`、Bootloader 信息更适合放到机型启动和内核差异分析里
