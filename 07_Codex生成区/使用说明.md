# Codex生成区使用说明

## 1. 目录定位

`07_Codex生成区` 用于集中存放以下内容：

- 让我生成的临时文档
- 汇总稿
- 对比稿
- 导出稿
- 临时结构化产物
- 截图、图片、排查材料整理结果

这个目录不是模块主文档层，也不是规则主文档层。  
它更适合作为：

- Codex 协作产物暂存区
- 过程材料整理区
- 后续再决定是否下沉到正式文档层的中转区

---

## 2. 当前使用规则

后续如果你让我生成以下内容，优先放这里：

- 临时整理结果
- 迁移映射清单
- 截图分类整理
- 说明性草稿
- 尚未正式进入模块层 / 规则层 / 需求层的辅助文件

如果后续某份内容已经稳定，再根据需要下沉到：

- `01_共享底座核心层`
- `02_模块文档`
- `03_规则文档`
- `04_需求文档`

---

## 3. 当前子目录建议

当前目录下建议优先按“产物类型”或“排查主题”分类，而不是继续把所有文件平铺在根目录。

例如：

- `图片与截图`
- `临时汇总`
- `导出结果`
- `对比稿`

如果是截图，继续按主题拆分，而不是继续堆在同一层。

## 5. 归位要求

从当前开始，`07_Codex生成区` 不只是“可选目录”，而是 Codex 生成产物的默认归位区之一。

也就是说：

- 让我生成的整理稿，不应先散落在根目录
- 让我生成的截图整理结果，不应先散落在根目录
- 让我生成的导出材料、说明稿、对比稿，应优先直接进入这个目录

如果后续某份内容已经稳定，再从这里下沉到正式文档层。
